## 1. 系統架構概覽

### 1.1 角色定義

| 角色 | 說明 |
|------|------|
| **Creator（創作者）** | 上傳並販售貼圖的個人或團體 |
| **User（用戶）** | 在 Vine App 內購買並使用貼圖的一般用戶 |
| **Platform（平台）** | Vine 營運方，負責審核、收款、分潤、商店展示 |

### 1.2 核心流程鳥瞰

```
[Creator]
  → 上傳素材 + 填寫商品資訊 + 設定售價
  → 提交審核
  → 審核通過後自動上架

[User]
  → 瀏覽 Vine 貼圖商店
  → 選擇貼圖組 → 直接付款（信用卡 / 各地 APM）
  → 付款成功 → 貼圖解鎖，可在 Vine 聊天室使用

[Platform]
  → Hyperswitch 收款入帳
  → 月底結算，扣除平台抽成後
  → 次月批次 Payout 給各創作者
```

### 1.3 技術邊界說明

- 貼圖**只在 Vine 自家 App 聊天功能**內使用，不支援匯出至第三方 App
- 付款走 **Hyperswitch**，不走 Apple IAP / Google Play Billing
- 購買紀錄與貼圖授權綁定 **Vine 帳號**（非裝置）

> ⚠️ **iOS 合規風險**：繞過 Apple IAP 在 iOS App 內進行數位內容購買，目前在非 EU 地區仍屬 Apple 政策灰色地帶。建議法務團隊在 App 上架前確認當前政策，或初期以 Web 版（mobile web）為主要購買入口。

## 2. 貼圖素材規格

### 2.1 靜態貼圖（Static）

| 規格項目 | Vine 規範 |
|----------|-----------|
| 格式 | PNG（透明背景，RGBA） |
| 色彩模式 | RGB |
| 解析度 | 最低 72 dpi |
| 尺寸 | 最大 **370 × 320 px**，最小 **80 × 80 px** |
| 像素限制 | 長寬均須為**偶數**像素 |
| 內距留白 | 裁切後與邊緣保留約 **10 px** 空白 |
| 檔案大小上限 | 單張 **500 KB** |

### 2.2 動態貼圖（Animated）

| 規格項目 | Vine 規範 |
|----------|-----------|
| 格式 | **APNG** 或 **WebP（動態）** |
| 尺寸 | 最大 **320 × 270 px** |
| 最短一邊 | ≥ 270 px |
| 播放長度 | 單次播放 ≤ **4 秒** |
| 迴圈次數 | 1–4 次 |
| 首幀要求 | 首幀必須能獨立傳達情緒（作為靜態預覽圖） |
| 檔案大小上限 | 單張 **800 KB** |

### 2.3 貼圖組數量

| 類型 | 可選數量 |
|------|----------|
| 靜態 | 8 / 16 / 24 / 32 / 40 張 |
| 動態 | 8 / 16 / 24 張 |

> 數量一旦提交審核後不可變更。

### 2.4 必交付素材清單

| 素材 | 規格說明 |
|------|----------|
| 貼圖圖檔 | 依選定數量，ZIP 打包上傳 |
| 封面主圖（Cover） | 代表整組的展示圖，從貼圖組中選取，建議正方形裁切版本 |
| 聊天室縮圖（Tab Icon） | 60 × 60 px，從貼圖組中選取 |
| 貼圖組名稱 | 1–40 字，支援多語系 |
| 商品描述 | 100 字以內，說明風格與適用情境 |
| 標籤（Tags） | 最多 10 個，用於搜尋排序 |
| 授權聲明 | 確認為原創或已取得授權 |

### 2.5 命名規則（ZIP 內部結構）

```
submission.zip
├── png/
│   ├── 01.png
│   ├── 02.png
│   └── ...
├── cover.png        ← 封面主圖
└── tab_icon.png     ← 聊天室縮圖
```

## 3. 創作者帳號與身份驗證

### 3.1 註冊前提

- 須擁有 Vine 用戶帳號
- 創作者申請為獨立流程（Creator Dashboard 入口）

### 3.2 身份驗證（KYC）

全球市場需符合各地 AML/KYC 法規，建議分兩階段：

| 階段 | 觸發條件 | 驗證內容 |
|------|----------|----------|
| **Tier 1（基本）** | 帳號申請時 | Email 驗證、居住國家聲明 |
| **Tier 2（收款）** | 首次申請 Payout 前 | 政府核發 ID（護照/身份證）、銀行帳戶或收款方式驗證 |

- 個人 vs 法人走不同的文件清單
- 高風險國家清單需額外人工審核

### 3.3 創作者後台功能

| 功能 | 說明 |
|------|------|
| 貼圖管理 | 上傳、查看審核狀態、上下架 |
| 銷售報表 | 每月銷售件數、收入金額 |
| 分潤查詢 | 累積分潤餘額、歷次 Payout 紀錄 |
| Payout 申請 | 填寫／更新收款資訊、查看申請狀態 |
| 帳號設定 | 個人資料、KYC 文件、稅務資訊 |

## 4. 貼圖上傳與提交流程

```
① 創作者進入 Creator Dashboard → 點擊「新增貼圖組」
          ↓
② 填寫基本資訊
   - 貼圖組名稱（多語系）
   - 商品描述
   - 標籤
   - 貼圖類型（靜態 / 動態）
   - 數量選擇（8 / 16 / 24 / 32 / 40）
          ↓
③ 上傳 ZIP 檔案
   - 系統自動驗證格式、尺寸、像素偶數、檔案大小
   - 即時回傳錯誤清單（逐張提示）
          ↓
④ 設定售價
   - 輸入金額（限制範圍見第 6 章）
   - 選擇銷售幣別（建議：USD 為基準，系統自動換算其他幣別）
          ↓
⑤ 確認授權聲明（勾選同意）
          ↓
⑥ 提交審核 → 進入審核佇列
```

## 5. 審核流程

### 5.1 流程狀態機

```
Draft（草稿）
  → [提交] → In Review（審核中）
                → [通過] → Approved（已核准）
                             → [創作者點擊上架] → On Sale（販售中）
                → [拒絕] → Rejected（已拒絕）
                             → [創作者修改] → Draft（草稿）

On Sale
  → [創作者主動下架] → Unlisted（已下架）
  → [平台強制下架]   → Removed（違規下架）
```

### 5.2 審核目標 SLA

| 類型 | 目標時間 |
|------|----------|
| 一般貼圖（AI 初審通過） | **3 個工作天**內完成 |
| AI 低信心分數（人工複審） | **7 個工作天**內完成 |
| 申訴案件 | **5 個工作天**內回覆 |

### 5.3 審核標準（六大類）

#### 一、圖像品質

- 不符合格式規範（尺寸、格式、像素奇數）
- 可視性差（過細、過長、全身像）
- 整組缺乏變化（全白、全數字、高度相似）
- 僅有 Logo 或純文字
- 含有錯字（英文拼寫、中文用字）
- 封面與實際貼圖內容嚴重不符
- 與已上架內容高度重複

#### 二、文字內容

- 貼圖組名稱、描述、標籤含有不當語言
- 標題與描述內容矛盾

#### 三、違禁內容（安全）

- 未成年人物涉及性暗示或暴力
- 露骨性描寫或血腥暴力
- 仇恨言論、歧視性符號（納粹、種族貶義等）
- 恐怖主義、極端主義相關視覺元素
- 令人不安的反社會元素

#### 四、商業限制

- 含有競品 App、通訊服務名稱或角色
- 企業廣告、促銷、募款用途
- 要求用戶提供個資才能使用

#### 五、版權與智慧財產

- 未授權使用他人 IP 角色（卡通、遊戲角色等）
- 侵害商標、著作權、肖像權
- 使用名人照片未取得授權

#### 六、技術合規

- 動態貼圖首幀無法傳達情緒
- 音效品質不符（過大聲、雜音、格式錯誤）

### 5.4 拒絕通知規範

絕通知須包含：

- 拒絕分類（從上方六大類中選取）
- 具體說明（指出哪張圖 / 哪個問題）
- 修改建議（非強制，但提供會大幅減少重工）
- 申訴入口連結

### 5.5 申訴機制

- 每次提交享有 **1 次申訴**機會
- 申訴須附上說明文字，平台由人工複審
- 申訴結果為最終決定

### 5.6 審核後內容限制

| 狀態 | 圖片 | 文字資訊 | 售價 |
|------|------|----------|------|
| 審核中 | ❌ 不可修改 | ❌ 不可修改 | ❌ 不可修改 |
| 販售中 | ❌ 不可修改 | ✅ 可修改（改後不需重審） | ✅ 可修改 |
| 下架後重新上傳 | ✅ 需重新提交審核 | ✅ | ✅ |

## 6. 定價機制

### 6.1 定價模式

**創作者自由定價**，平台設定全球通用的上下限：

| 項目 | 建議值 |
|------|--------|
| 最低售價 | **USD $0.99** |
| 最高售價 | **USD $29.99** |
| 定價單位 | 以 USD 為基準輸入，系統自動換算其他幣別 |
| 換算幣別 | 依 Hyperswitch 支援的幣別動態換算 |
| 換算更新頻率 | 每日更新匯率 |

> 上下限數值為建議值，正式上線前應依市場調研確定。

### 6.2 幣別顯示邏輯

- 用戶看到的售價為**本地幣別**（依帳號 / IP 判斷）
- 結帳頁面顯示本地幣別金額與匯率來源
- 創作者後台報表以 **USD** 為統一結算幣別

### 6.3 調價規則

- 創作者可隨時修改售價，**即時生效**
- 已購買的用戶不受調價影響（購買當下鎖定價格）
- 不支援「降價通知」推播（未來可作為進階功能）

## 7. 購買流程（用戶側）

### 7.1 流程設計

```
① 用戶在 Vine 聊天室 / 商店頁面瀏覽貼圖
          ↓
② 點擊貼圖組 → 進入商品詳情頁
   - 封面預覽、全組縮圖預覽
   - 創作者名稱（可點擊進入創作者頁）
   - 售價（本地幣別）
   - 購買 CTA 按鈕
          ↓
③ 點擊「購買」→ 跳出結帳流程（Hyperswitch Checkout）
   - 支援儲存的付款方式（回購用戶一鍵結帳）
   - 支援多種 APM（依用戶地區顯示適合的支付方式）
          ↓
④ 付款成功
   - 即時解鎖貼圖（綁定 Vine 帳號）
   - 聊天室貼圖列表自動更新
   - 推播 / 站內通知：「已解鎖 XXX 貼圖組！」
          ↓
⑤ 付款失敗
   - 顯示失敗原因（Hyperswitch 回傳錯誤碼轉譯）
   - 提供重試入口
```

### 7.2 購買狀態說明

| 狀態 | 說明 |
|------|------|
| 未購買 | 商品詳情頁顯示售價與購買按鈕 |
| 已購買 | 商品詳情頁顯示「已擁有」，聊天室可使用 |
| 貼圖被下架 | 已購買用戶可繼續使用，但商店不再展示 |

### 7.3 退款政策

- 數位內容購買原則上**不提供退款**
- 例外：技術性錯誤（如付款成功但貼圖未解鎖）→ 系統自動補發或退款
- 爭議案件走 Hyperswitch 的 Chargeback 流程

## 8. 支付層：Hyperswitch 整合

### 8.1 為何選擇 Hyperswitch

- **開源** (Apache 2.0)，可自部署，無 vendor lock-in
- 單一 API 接入 50+ PSP（Stripe、Adyen、Braintree 等）
- 支援 75+ 種支付方式（信用卡、Apple Pay、Google Pay、BNPL、UPI、本地 APM）
- 內建智能路由（依地區、失敗率、成本自動選擇最佳 PSP）
- PCI DSS 合規，自帶 Token Vault

### 8.2 全球支付方式矩陣（主要地區）

| 地區 | 主要支援方式 |
|------|-------------|
| 台灣 | 信用卡、LINE Pay（透過 PSP）、街口支付 |
| 日本 | 信用卡、コンビニ払い、PayPay |
| 東南亞 | GrabPay、GoPay、PromptPay、OVO |
| 歐洲 | 信用卡、SEPA、iDEAL、Klarna |
| 美國 | 信用卡、Apple Pay、Google Pay、PayPal |
| 印度 | UPI、NetBanking、Paytm |

> 確切支援清單以 Hyperswitch 最新 connector 列表為準。

### 8.3 智能路由策略建議

- 主要路由：依用戶地區選擇成本最低的 PSP
- 備援路由：主 PSP 失敗時自動 failover 至次要 PSP
- 重試策略：特定錯誤碼（如暫時性網路錯誤）自動重試，最多 2 次

### 8.4 Payout 層

Phase 2.5 的創作者出款先採 **manual payout**。Hyperswitch 仍是用戶付款的收單層,但不作為初版創作者打款執行層。

初版出款執行方式：

- 系統建立月結 ledger 與 payout request
- 營運審核後建立 payout batch,鎖定該批次金額
- 後台匯出 CSV,交由營運在銀行或金流後台人工匯款
- 匯款完成後由營運回填銀行交易序號、付款日期、成功 / 失敗原因
- 創作者收到站內通知與 Email,並可在 Dashboard 查看狀態

> 未來升級路徑：若 hyperswitch-prism 後續完整支援 Stripe payouts,可在同一個 batch / ledger 架構下新增 automated payout executor。Manual executor 產生 CSV,automated executor 呼叫 hyperswitch-prism `PayoutClient` 或 Stripe Connect API,避免重寫創作者端流程。

## 9. 創作者分潤與 Manual Payout 機制

### 9.1 分潤結構

```
用戶付款金額（真實貨幣）
  - 支付手續費（約 1.5%–3%，依 PSP 而定，由平台吸收或轉嫁，待決策）
  = 淨收入（Net Revenue）
  × 創作者分潤比例（待定，建議 70%）
  = 創作者應得分潤
  - 預扣所得稅（依創作者居住地稅率）
  = 創作者實際到手金額
```

> **建議分潤比例 70%**：LINE 因走 IAP 只能給 35%，Vine 直收可給更高比例，這是重要的創作者招募競爭優勢。最終比例依財務模型確認。

### 9.2 月結流程

| 時程 | 動作 |
|------|------|
| 每月最後一天 | 系統結算當月所有已完成訂單 |
| 次月 5 日前 | 計算各創作者分潤，寄發結算報告 Email |
| 次月 10 日 | 創作者於 Dashboard 確認金額並申請 Payout |
| 次月 15–20 日 | 營運審核 payout request,建立批次,下載 CSV,人工匯款 |
| 打款完成 | 營運回填結果,系統發送站內通知 + Email 通知 |

### 9.3 最低 Payout 門檻

| 項目 | 建議值 |
|------|--------|
| 最低申請門檻 | 產品目標 **USD $10 等值**；Phase 2.5 TWD 初版用 **NT$300** |
| 未達門檻 | 累積至次月，不自動打款 |
| 最高單次打款上限 | **USD $10,000**（超過需人工審核）|

### 9.4 支援的收款方式

Phase 2.5 初版只支援銀行轉帳。

創作者需在 Tier 2 KYC 填寫：

- 收款人法定姓名 / 戶名
- 銀行代碼、銀行名稱、分行資訊
- 銀行帳號
- 收款幣別與居住地

創作者可在收款申請頁新增或修改銀行轉帳資料。每次修改會停用舊的 active account 並建立新的 active account；已建立的 payout request 仍保留當時選用的 `payoutAccountId`。完整銀行帳號只存於 server/private 欄位與營運 CSV 匯出檔,前台只顯示銀行名稱與末四碼,且儲存後不回傳完整帳號。PayPal 或其他 payout method 暫不列入 Phase 2.5,避免在人工匯款流程中同時支援多套營運規則。

### 9.4.1 Payout request 狀態

| 狀態 | 說明 |
|------|------|
| available | 已結算且達最低門檻,可申請 |
| requested | 創作者已送出申請,等待營運審核 |
| approved | 營運已核准,等待建立批次 |
| exported | 已納入 payout batch 並匯出 CSV,金額鎖定 |
| paid | 營運已回填匯款完成 |
| rejected | 審核退件,需補資料或重新申請 |
| failed | 銀行匯款失敗,需更新收款資訊後重排批次 |

### 9.4.2 Payout batch 與 CSV 匯出

每個 payout batch 需記錄 `batchId`、建立者、建立時間、匯出時間、匯出者、付款日期、狀態與 audit log。CSV 至少包含：

- batch ID、payout request ID、creator ID、creator display name
- 收款人法定姓名、銀行代碼、銀行名稱、分行、完整銀行帳號、帳號末四碼
- 幣別、gross amount、tax withholding、transfer fee、net amount
- internal reference、memo / notes

### 9.5 稅務處理

- 創作者需在 Tier 2 KYC 時填寫稅務資訊（居住地、稅籍號碼）
- 平台依各國稅率自動計算預扣稅額
- 每年提供年度收入憑證供創作者報稅使用
- 平台不提供稅務建議，建議創作者自行諮詢當地稅務師

### 9.6 分潤異常處理

| 異常情境 | 處理方式 |
|----------|----------|
| 用戶退款（Chargeback） | 從該創作者下月分潤中扣除 |
| 疑似刷量購買 | 凍結該筆分潤，人工調查 |
| Payout 審核退件 | 營運標記 rejected,填寫原因,通知創作者補資料後重新申請 |
| 人工匯款失敗 | 營運標記 failed,填寫銀行退件原因,通知創作者更新收款資訊後重排批次 |
| 帳號封鎖 | 凍結未打款分潤，待申訴解除後釋出 |

## 10. 貼圖授權與使用機制（App 內）

### 10.1 授權綁定

- 購買後授權綁定至 **Vine 帳號**（非裝置）
- 同一帳號多裝置登入均可使用
- 不可轉讓、不可退款（技術錯誤除外）

### 10.2 聊天室整合

- 聊天室下方貼圖列表顯示「**已擁有**」的貼圖組
- 預覽模式：未購買的貼圖組可在商店頁查看縮圖，無法在聊天室插入
- 用戶可為常用貼圖設定「收藏 / 最愛」排序

### 10.3 貼圖訊息格式（LINE Messaging API Compatible）

Vine 的聊天室貼圖訊息格式設計為相容 LINE Messaging API 的 sticker message object，方便未來與 LINE Bot 生態系整合或對接第三方工具。

#### 10.3.1 識別碼系統

| 欄位 | 說明 | 對應 LINE 欄位 |
|------|------|----------------|
| `packageId` | 貼圖組 ID，全平台唯一字串 | `packageId` (String) |
| `stickerId` | 單張貼圖 ID，組內唯一 | `stickerId` (String) |
| `stickerResourceType` | 貼圖資源類型（見下表）| `stickerResourceType` (String) |
| `keywords` | 最多 15 個描述關鍵字，用於搜尋與情境推薦 | `keywords` (Array of strings) |

> `packageId` 與 `stickerId` 均使用 **String 型別**（與 LINE 一致），內部實作建議使用 UUID 或自增 ID 字串化。

#### 10.3.2 stickerResourceType 枚舉

| 值 | 說明 | 對應 Vine 貼圖類型 |
|----|------|--------------------|
| `STATIC` | 靜態貼圖 | 靜態 PNG |
| `ANIMATION` | 動態貼圖（無聲）| 動態 APNG / WebP |
| `SOUND` | 有聲靜態貼圖 | 靜態 + 音效 |
| `ANIMATION_SOUND` | 有聲動態貼圖 | 動態 + 音效 |
| `POPUP` | 彈出貼圖 / 特效貼圖 | 全螢幕動畫 |
| `POPUP_SOUND` | 有聲彈出貼圖 | 全螢幕動畫 + 音效 |
| `CUSTOM` | 客製貼圖（不回傳輸入文字）| 訊息貼圖（基礎型）|
| `MESSAGE` | 訊息貼圖（可回傳用戶輸入文字）| 訊息貼圖（進階型）|

> ⚠️ 依 LINE 官方慣例，未來可能新增 type 而不預告，實作時應使用 **fallback 機制**，遇到未知 type 一律以 `STATIC` 顯示，不拋例外。

#### 10.3.3 聊天室訊息 Payload 格式

**傳送方（Sender）送出貼圖時，chat message 的 sticker 欄位：**

```json
{
  "type": "sticker",
  "packageId": "vine_pkg_abc123",
  "stickerId": "vine_stk_001"
}
```

**接收方（Receiver）從 Vine 伺服器收到的完整 sticker message object（Webhook / WS 推送）：**

```json
{
  "type": "sticker",
  "id": "vine_msg_987654321",
  "packageId": "vine_pkg_abc123",
  "stickerId": "vine_stk_001",
  "stickerResourceType": "ANIMATION",
  "keywords": ["開心", "慶祝", "派對", "happy", "celebrate"],
  "cdnUrl": "https://cdn.vine.app/stickers/vine_pkg_abc123/vine_stk_001.png",
  "previewUrl": "https://cdn.vine.app/stickers/vine_pkg_abc123/vine_stk_001_preview.png"
}
```

> `cdnUrl` 與 `previewUrl` 為 Vine 自行擴充欄位（LINE 沒有），讓 App 端可直接使用 URL 渲染，不需再查一次 API。

#### 10.3.4 CDN 存取權限設計

| 情境 | 存取方式 | 說明 |
|------|----------|------|
| **傳送貼圖** | 需已購買 → App 端驗證 | 聊天室插入時由 Vine App 前端進行授權檢查 |
| **顯示收到的貼圖** | CDN 公開讀取，**不需購買** | 接收方無論是否購買，均可正常顯示對方傳來的貼圖 |
| **商店縮圖預覽** | CDN 公開讀取 | 縮圖（`previewUrl`）全公開，方便搜尋引擎索引與商店展示 |
| **原始高解析圖檔** | CDN 公開讀取 | 因已購買才能傳送，顯示時無需再驗證 |

> 這個設計與 LINE 的行為一致：**傳送是付費門檻，顯示是免費行為**，確保聊天室不會出現破圖。

#### 10.3.5 圖檔 CDN URL 命名規則

```
https://cdn.vine.app/stickers/{packageId}/{stickerId}.png          ← 原始圖
https://cdn.vine.app/stickers/{packageId}/{stickerId}_preview.png  ← 縮圖（Tab Icon 尺寸）
https://cdn.vine.app/stickers/{packageId}/cover.png                ← 封面主圖
```

- 所有路徑均為靜態 URL，無需 token（CDN 公開讀取）
- 動態貼圖副檔名依格式：APNG → `.png`，WebP → `.webp`
- 有聲貼圖音效：`https://cdn.vine.app/stickers/{packageId}/{stickerId}.m4a`

#### 10.3.6 版本管理

- 貼圖上架後圖檔**不可修改**（與 LINE 一致），因此不需要版本號機制
- 若未來允許修改（例如小幅修圖），應在 URL 加入版本號：`{stickerId}_v2.png`，以防 CDN cache 污染

### 10.4 下架貼圖的處理

- 貼圖被下架（無論主動或違規）：**已購買用戶仍可繼續使用**
- 未購買用戶無法再購買，商店不再展示
- 若下架原因為版權侵權：視情況強制停用，並退款給已購買用戶

## 13. 版權與內容法律框架

### 13.1 創作者聲明

創作者提交貼圖時需勾選同意以下聲明：

- 內容為本人原創，或已取得合法授權
- 未侵害任何第三方商標、著作權、肖像權
- 授權 Vine 平台在服務範圍內使用（展示、行銷素材）
- 了解違反規範將導致下架及帳號處分

### 13.2 平台授權範圍

- Vine 取得非獨家授權，用於：商店展示、行銷推廣、縮圖生成
- Vine **不主張**貼圖著作權歸屬，創作者保留所有權
- Vine 可在官方行銷素材中使用貼圖圖片（需事先告知創作者）

### 13.3 侵權投訴流程（DMCA / 類 DMCA）

```
① 權利人提交侵權投訴（指定表單，需提供：著作權聲明、侵權貼圖連結、聯絡方式）
          ↓
② 平台收到後 48 小時內暫時下架該貼圖組
          ↓
③ 通知創作者，給予 7 天提出反駁（Counter Notice）
          ↓
    ┌─────────────────┐
    │ 創作者提出反駁    │  → 平台通知投訴方，14 天無進一步行動 → 恢復上架
    │ 創作者未回應      │  → 永久下架，退款已購買用戶
    └─────────────────┘
```

## 14. 平台治理與違規處理

### 14.1 違規處分層級

| 層級 | 觸發條件 | 處分 |
|------|----------|------|
| **警告** | 輕微違規（如描述含輕微不當用詞）| 通知修改，限期 3 天 |
| **下架** | 內容違規（審核標準 3–6 類）| 貼圖組下架，可申訴 |
| **帳號暫停** | 多次違規 / 嚴重侵權 | 停止上架與收款能力，保留帳號 30 天申訴期 |
| **帳號終止** | 詐欺、嚴重違法 | 永久封鎖，凍結分潤餘額 |

### 14.2 申訴流程

- 帳號暫停後 30 天內可提出申訴
- 需附上說明文件
- 平台 **5 個工作天**內回覆結果
- 申訴結果為最終決定

### 14.3 分潤凍結規則

- 帳號暫停期間：分潤累積但不可提領
- 帳號解除暫停：凍結分潤釋出，可正常提領
- 帳號終止：凍結分潤依各地法規處理（通常 90 天後沒收）
