## 1. 平台概覽

| 項目 | LINE Creators Market 現況 |
|------|--------------------------|
| 上線時間 | 2014 年 5 月（貼圖）；2016 年 4 月（主題） |
| 創作者規模 | 全球超過 610,000 名註冊創作者 |
| 銷售地區 | 最多 230 個國家與地區 |
| 販售品項 | 靜態貼圖、動態貼圖、表情符號、主題、訊息貼圖、彈出貼圖、特效貼圖 |
| 平台入口 | LINE App 貼圖商店、LINE STORE（網頁版）|

**核心商業邏輯**：創作者免費上傳 → 平台免費審核 → 上架後按銷售分潤 → 平台抽成後分給創作者 35%（扣除 Apple/Google 30% 後再五五分）。

## 2. 貼圖素材規格

### 2.1 靜態貼圖（Static Stickers）

| 規格項目 | 規範內容 |
|----------|----------|
| 檔案格式 | PNG（透明背景） |
| 色彩模式 | RGB |
| 解析度 | 最低 72 dpi |
| 尺寸上限 | 370 × 320 像素 |
| 尺寸下限 | 80 × 80 像素 |
| 像素規則 | 長寬均須為**偶數**像素 |
| 內距（Margin） | 裁切後內容與邊緣保留約 **10 pixels** 留白 |
| 主檔建議解析度 | 740 × 640 像素（供平台行銷素材使用） |
| 命名規則 | `01.png`、`02.png`…以兩位數命名 |
| 資料夾命名 | `png`（全小寫） |
| 壓縮方式 | 打包為 `.zip` 上傳 |

### 2.2 動態貼圖（Animated Stickers，APNG）

| 規格項目 | 規範內容 |
|----------|----------|
| 檔案格式 | APNG |
| 尺寸上限 | 320 × 270 像素 |
| 至少一邊 | ≥ 270 像素 |
| 播放迴圈 | 1–4 次，總播放時間 **≤ 4 秒** |
| 首幀要求 | 第一幀需能獨立傳達情緒（靜態預覽用） |
| 資料夾命名 | `animation`（全小寫） |
| 檔案大小 | 單檔有上限（官方未公開，建議控制在 300KB 以下） |

### 2.3 有聲貼圖（Sound Stickers）

| 規格項目 | 規範內容 |
|----------|----------|
| 音效格式 | M4A |
| 命名規則 | `01.m4a`…兩位數；主圖音效為 `main_sound.m4a` |
| 資料夾命名 | `sound`（全小寫） |
| 音質要求 | 不可過大聲、過小聲、含過多環境雜音 |
| 稽核工具 | LINE 使用 Adobe Audition 檢查 |
| DRM | 不加密，無 DRM 保護 |

### 2.4 貼圖組數量選擇

| 類型 | 可選數量 |
|------|----------|
| 靜態貼圖 | 8 / 16 / 24 / 32 / 40 張 |
| 動態貼圖 | 8 / 16 / 24 張 |

> 注意：上架中或審核中的貼圖組，數量不可修改。

### 2.5 必交付素材清單

| 素材 | 規格說明 |
|------|----------|
| 貼圖圖片 | 依選定數量上傳 |
| 主圖（Main Image） | 代表整組的封面圖，需從貼圖組中選取 |
| 聊天室 Tab 縮圖 | 小尺寸識別用，需從貼圖組中選取 |
| 標題（Title） | 最多 40 字（含多語系） |
| 商品說明（Description） | 介紹貼圖組用途與風格 |
| 版權資訊 | 著作人/品牌聲明 |
| 語言設定 | 可設定對應販售地區語言 |

### 2.6 彈出貼圖（Pop-up Stickers）

- 動畫播放範圍：**全螢幕**
- 尺寸會依用戶裝置自動縮放
- 需在設定表單中指定位置

### 2.7 特效貼圖（Effect Stickers）

- 動畫播放於**聊天室背景**
- 其餘規格同動態貼圖

## 3. 審核流程

### 3.1 流程概覽

```
創作者上傳素材
      ↓
填寫商品資訊（標題、描述、版權、語言、價格）
      ↓
提交審核（Submit for Review）
      ↓
LINE 人工 + 系統審核（最長 30 個工作天）
      ↓
    ┌──────┴──────┐
  核准（Approved）  拒絕（Rejected）
      ↓                  ↓
創作者點擊「上架」     收到拒絕原因
（Release Button）      ↓
      ↓             修改後重新提交
正式販售
```

### 3.2 審核標準（六大類）

#### 類別一：圖像品質
- 不符合平台指定格式
- 不易用於日常對話（實用性不足）
- 可視性差（過長、全身像等）
- 整組缺乏變化（全為淡色、數字排列等）
- 僅包含 Logo、純文字或短句
- 含有拼寫錯誤
- 標題與說明內容相互矛盾
- 主圖與實際貼圖內容不符
- 與已上架或審核中的貼圖重複

#### 類別二：文字內容規範
- 創作者名稱、標題、說明中含有不當文字
- 亞洲語言文字（中、日、韓等）每個字計為 2 個字元

#### 類別三：違禁內容（內容安全）
- 未成年飲酒、吸菸
- 露骨性暗示或暴力
- 仇恨、歧視性內容
- 反社會行為描繪
- 令用戶不安的反社會元素

#### 類別四：商業與廣告
- 要求用戶提供個資才能購買
- 超出私人使用範疇（企業廣告促銷用途）
- 含有競品 App 或通訊服務之名稱或角色
- 募款、捐款、政治或宗教宣傳
- 求職廣告等商業廣告

#### 類別五：版權與法律
- 侵害 LINE 或第三方之商標、著作權、專利權
- 使用未獲授權之名人照片、IP 角色
- 照片類貼圖需提供使用授權証明文件

#### 類別六：地區限制
- 部分貼圖依內容、地區或創作者特性，可能僅核准特定地區販售

### 3.3 審核後的內容限制

| 狀態 | 可修改項目 |
|------|-----------|
| 審核中 | 可下架撤回，不可修改圖片 |
| 已上架 | **僅可修改文字資訊**，圖片無法修改 |
| 下架修改 | 修改後須重新提交審核 |

## 4. 定價與幣別機制

### 4.1 定價方式

- 以 **LINE Coins**（平台虛擬貨幣）計價
- 最低售價：**50 Coins**
- 日本地區另可以日圓定價（最低 JPY 120）
- 其他地區依匯率換算對等價格

### 4.2 平台費用拆分

```
用戶購買金額（含平台幣）
    ↓ 扣除 Apple App Store / Google Play 手續費 30%
剩餘 70%
    ↓ 平台（LINE）與創作者五五分
創作者實得：約 35% of 總收入
    ↓ 再依居住地扣除預扣所得稅
創作者最終入帳金額
```

> 注意：Apple/Google 手續費僅針對透過 App 內購的部分；LINE Coins 購買管道手續費結構可能略有不同。

## 5. 創作者收款機制

### 5.1 收款流程

1. 每月最後一天統計當月銷售
2. 次月中旬確認分潤金額，LINE 官方帳號推播通知
3. 創作者至帳號設定確認「轉帳資訊」已填寫
4. 於「收款申請頁面」確認金額並提交申請
5. 提交申請後 **90 天內**完成轉帳
6. 轉帳完成後 LINE 官方帳號推播通知

### 5.2 最低提領門檻

- **JPY 1,000**（或當地貨幣等值金額）
- 未達門檻則累積至次月

### 5.3 支援的轉帳方式（依地區）

| 方式 | 說明 |
|------|------|
| 銀行轉帳 | 需填寫銀行帳號；有固定轉帳手續費，依國家不同（如泰國 THB 50） |
| LINE Pay 電子錢包 | 轉入 LINE Money；無提領手續費（截至 2016 年資料） |
| ~~PayPal~~ | 2022 年 11 月起停止支援 |

### 5.4 扣款項目

- 轉帳手續費（由創作者負擔，從分潤中扣除）
- 外匯手續費（跨幣別時，匯率由 LINE 自行決定）
- 預扣所得稅（依創作者與購買者居住地稅率計算）
- 提領上限：**JPY 1,000,000 / 次**

### 5.5 稅務申報

- 創作者需提交「所得稅協定申請書」（Tax Convention Form）
- 提交後帳號狀態從「未提交」更新為「已提交」
- 適用的預扣稅率依雙邊租稅協定而異

## 6. 創作者帳號與身份驗證

### 6.1 註冊前提

- 需擁有 LINE 帳號（手機 App 建立）
- 完成創作者帳號註冊（Creators Market 網站）

### 6.2 帳號資料

- 創作者名稱（對外顯示）
- 基本資訊（姓名、居住地、聯絡方式）
- 轉帳資訊（銀行帳戶或 LINE Pay）
- 稅務資訊

### 6.3 帳號停用與分潤處理

- 帳號刪除時，若分潤餘額超過最低門檻，90 天內仍會轉帳
- 以下情況**分潤沒收**：
  - 基本資訊或轉帳資訊未填寫或填寫錯誤
  - 未完成必要驗證
  - 超過一年無法聯繫到創作者
  - 服務終止時分潤未達最低門檻

### 6.4 法人 vs 個人創作者

- 個人與法人（企業）皆可申請
- 稅務文件要求不同

## 8. 訂閱制模式（Premium）

LINE Stickers Premium 是 LINE 的訂閱制分潤補充模式：

- 用戶繳交月費 → 可無限使用訂閱範圍內的貼圖
- 創作者依「該月該貼圖組的使用量占比」分得訂閱池分潤
- 創作者可自行選擇是否加入 Premium 方案
- 加入後可隨時退出（退出生效時間為次月）

**收益公式（概念）**：
```
創作者月分潤 = 訂閱分潤池總額 × (該創作者貼圖使用次數 / 全平台總使用次數)
```

## 9. 版權與法律框架

### 9.1 原創性要求

- 所有提交內容必須為創作者**原創**
- 使用照片須提供使用授權證明

### 9.2 授權範圍

- 創作者上架即授權 LINE 在平台上行銷、推廣、展示
- LINE 可使用貼圖圖片製作橫幅廣告等行銷素材
- 創作者保留著作權，但授予平台非獨家使用權

### 9.3 禁止內容（知識產權面）

- 侵害他人商標、著作權、專利
- 未授權使用名人肖像
- 使用其他 App（如 WhatsApp、WeChat）的角色或品牌

### 9.4 協同創作（Collaboration Contents）

- LINE 可指定特定第三方 IP 開放創作者進行衍生創作
- 衍生作品需符合 LINE 額外規定
- 此類內容由 LINE 判定是否構成合規衍生作品

### 9.5 申訴機制

- 創作者對於審核結果無明確申訴流程（LINE 目前以人工複審處理）

## 10. 商店與曝光機制

### 10.1 上架入口

- LINE App 內建貼圖商店
- LINE STORE 網頁版（store.line.me）

### 10.2 搜尋與標籤

- 創作者可在上傳後設定 **Tag（標籤）**
- 標籤影響搜尋排名與推薦演算法

### 10.3 語言與地區設定

- 標題與說明可設定多語系版本
- 可指定販售地區（最多 230 個國家）
- 系統依語言/地區自動分流展示

### 10.4 推薦曝光

- 平台自行決定首頁推薦、編輯精選（無透明機制）
- 新貼圖上架初期可獲得「新品曝光期」

## 12. 創作者溝通與通知機制

| 事件 | 通知方式 |
|------|---------|
| 分潤確認 | LINE 官方帳號推播 |
| 收款申請成功 | LINE 官方帳號推播 |
| 轉帳完成 | LINE 官方帳號推播 |
| 貼圖審核結果 | 帳號後台狀態更新（核准/拒絕） |
| 拒絕原因 | 帳號後台文字說明（有限） |
| 平台政策更新 | 官網公告 + Email（不定期） |

> **缺口**：LINE 的溝通機制相對被動，創作者對審核進度與拒絕原因透明度不高，這是自建系統可以優化的點。

## 13. 平台治理與違規處理

### 13.1 內容下架條件

- 上架後發現違反審核規範
- 收到有效的版權侵權投訴
- 涉及詐欺、不當購買行為

### 13.2 帳號封鎖條件

- 違反使用條款
- 提交虛假個人資訊
- 超過一年失聯

### 13.3 帳號申訴

- 目前 LINE 對帳號封鎖的申訴管道有限
- 創作者可透過 Help Center 表單聯繫

## 14. 自建系統差距分析與建議

以下為 PM 視角補充，LINE 未提供或不足之處，建議自建系統時納入規劃：

### 14.1 審核透明度提升
**現況問題**：LINE 審核結果說明過於簡略，拒絕原因不明確。
**建議**：
- 拒絕時提供結構化原因標籤（如「內容違規 > 未成年相關」）
- 提供審核進度即時狀態（已接收 / 審核中 / 完成）
- 建立申訴流程（限一次），有明確 SLA

### 14.2 創作者數據賦能
**現況問題**：LINE 只提供銷售數字，缺乏行銷洞察。
**建議**：
- 商品頁曝光次數、點擊率、加入購物車率
- 用戶地區與語言分布
- 貼圖被傳送次數（使用量）vs 購買次數

### 14.3 收款彈性
**現況問題**：LINE 主要依賴銀行轉帳，移除 PayPal 後選項變少。
**建議**：
- 支援更多本地化支付（台灣可支援：玉山銀行、街口、LINE Pay、歐付寶）
- 降低提領門檻或提供即時提領選項
- 支援加密貨幣（可選）

### 14.4 創作者成長計畫
**現況問題**：LINE 對創作者無分級制度。
**建議**：
- 建立創作者等級制度（新星 / 進階 / 精英）
- 依等級提供更高分潤比例、優先審核、官方推薦
- 每月/季「編輯精選」競賽，入選者額外獎勵

### 14.5 社群與互動機制
**現況問題**：LINE 是純交易平台，缺乏社群感。
**建議**：
- 創作者公開頁面（作品集 + 粉絲訂閱）
- 用戶可「追蹤創作者」，新品上架時收到通知
- 用戶評分與留言（但需嚴格審核防濫用）

### 14.6 多樣化商業模式
**現況問題**：LINE 只有單次購買 + Premium 訂閱兩種模式。
**建議**：

| 模式 | 說明 |
|------|------|
| 單次購買 | 基本款，一次性付費 |
| 訂閱制 | 月費制，無限使用特定分類 |
| 限時免費 | 創作者自行設定限時免費活動，換取曝光 |
| 打包販售 | 多組貼圖組合折扣 |
