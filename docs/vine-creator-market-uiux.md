**UX 說明**

- **搜尋列置頂**：LINE 的搜尋藏在右上角圖示內，造成發現性低。Vine 應將搜尋列顯眼置頂，降低探索門檻。
- **Horizontal Pill Filter**：用 type 分類 pill 替代 LINE 的階層式分類，減少點擊層數，一個手勢即可切換。
- **卡片資訊密度**：每張卡片只顯示封面、名稱、價格，不塞過多資訊。**動態貼圖卡片在 hover / 長按時自動播放預覽**，這是 LINE 桌面版沒有的體驗。
- **橫向捲動**：各區塊保持 4 張可見 + 半張露出（signal there's more），引導用戶橫滑探索。
- **價格顯示**：直接顯示真實貨幣（NT$ / USD），移除 LINE Coins 轉換的認知負擔，這是 Vine 最重要的 UX 差異之一。

### U2. 搜尋與篩選結果頁

```

│  ← 返回       🔍 "貓咪"            ✕        │
├─────────────────────────────────────────────┤
│  類型 ▾   價格 ▾   排序：熱門 ▾    篩選 ≡   │  ← 篩選列，下拉或 Sheet
├─────────────────────────────────────────────┤
│  找到 142 個結果                             │
├──────────────┬──────────────────────────────┤
│  [封面圖]    │  貓咪日常 40 款              │
│              │  by 喵星人工作室  ⭐ 4.8     │
│   動態       │  動態 · 40 張                │
│   BADGE      │  NT$ 75           [購買]     │
├──────────────┼──────────────────────────────┤
│  [封面圖]    │  懶貓系列                    │
│              │  by CatLover      ⭐ 4.6     │
│   靜態       │  靜態 · 24 張                │
│   BADGE      │  NT$ 45           [購買]     │

│  [封面圖]    │  貓咪上班日記                │
│              │  by officecat     ⭐ 4.9     │
│   有聲       │  有聲動態 · 16 張            │
│              │  NT$ 90      ✓ 已擁有        │  ← 已購用戶顯示「已擁有」
├──────────────┴──────────────────────────────┤
│  …                                          │
└─────────────────────────────────────────────┘
```

**UX 說明**

- **List 而非 Grid**：搜尋結果用 list layout，讓每筆結果有足夠空間顯示名稱、創作者、類型 badge、價格。Grid 在搜尋場景中難以比較資訊。
- **類型 Badge**：靜態／動態／有聲用顏色 badge 快速區分，不需點入才知道。
- **已擁有狀態**：結果頁直接標示「已擁有」，避免用戶重複點擊商品頁才發現已購買（LINE 的痛點）。
- **評分顯示**：LINE 沒有用戶評分。Vine 可加入評分作為信任信號，但早期可先隱藏，等用戶評論機制建立後再開放。
- **篩選設計**：「類型」「價格區間」「排序」分開控制，各自下拉，避免一個全域篩選 Modal 造成流程中斷。

- **Sticky Bottom Bar 購買 CTA**：價格 + 購買按鈕固定在螢幕底部，無論用戶滑到哪裡都可即時購買。LINE App 的購買按鈕需要滑到頁面特定位置才能點到。
- **動態貼圖自動播放封面**：商品詳情頁的封面大圖若為動態貼圖，應自動靜音播放，讓用戶直接感受動態效果，這是最強的轉換說服工具。
- **小縮圖橫向預覽**：讓用戶一眼看到整組貼圖的所有內容，點擊可放大預覽單張。LINE 在這個設計上做得較好，Vine 應保留。
- **評價區塊**：LINE 完全沒有評價機制，這是明顯缺口。Vine 加入評分與短評，對轉換率有正面影響，但要防止刷評。
- **創作者 Card**：商品頁內嵌入創作者小卡，包含追蹤按鈕，讓用戶自然流入創作者頁面，建立粉絲關係。
- **同創作者推薦**：頁面底部顯示同創作者其他作品，提升同捆購買率（LTV 提升）。

### U4. 結帳流程

> Vine 採用直接結帳（Direct Checkout）而非購物車，因為貼圖是單次衝動型購買，購物車會增加摩擦。

**Step 1 — 結帳 Modal（Bottom Sheet）**

```
┌─────────────────────────────────────────────┐
│  ▬                                          │  ← 可下拉關閉的 handle
│                                             │
│  確認購買                                   │
│  ─────────────────────────────────────────  │
│  [封面圖] 貓咪日常 40 款                    │
│           動態 · 40 張                      │
│           NT$ 75                            │
│                                             │
│  付款方式                                   │
│  ┌─────────────────────────────────────┐   │
│  │ ● 💳 Visa  ···· 4242       [更改]   │   │  ← 預設上次使用的付款方式
│  └─────────────────────────────────────┘   │
│  ┌─────────────────────────────────────┐   │
│  │    Apple Pay                        │   │  ← 快速支付選項
│  └─────────────────────────────────────┘   │
│  ┌─────────────────────────────────────┐   │
│  │    Google Pay                       │   │
│  └─────────────────────────────────────┘   │
│  + 使用其他付款方式                         │
│                                             │
│  小計         NT$ 75                        │
│  ─────────────────────────────────────────  │
│  ┌─────────────────────────────────────────┐│
│  │         確認付款 NT$ 75                 ││  ← 主 CTA，顯示金額再次確認
│  └─────────────────────────────────────────┘│
│                                             │
│  付款後即可在 Vine 聊天室使用此貼圖組        │  ← 消除疑慮的說明文字
└─────────────────────────────────────────────┘
```

**Step 2 — 付款處理中**

```
┌─────────────────────────────────────────────┐
│                                             │
│              ⏳                             │
│                                             │
│         處理付款中，請稍候...               │
│                                             │
│     請勿關閉此頁面                          │
│                                             │
└─────────────────────────────────────────────┘
```

**UX 說明**

- **Bottom Sheet 而非跳轉新頁**：結帳以 Bottom Sheet Modal 呈現，用戶仍可感受到商品詳情在背後，降低「我真的要離開商品頁？」的心理阻力。
- **預設上次付款方式**：回購用戶直接看到已存的卡片，一鍵 confirm 即完成，這是提升二次購買率的關鍵設計。
- **Apple Pay / Google Pay 置頂**：本地 wallet 支付是最快的方式（生物辨識 1 步完成），應優先展示。Hyperswitch 支援動態展示可用的 wallet。
- **CTA 顯示金額**：按鈕文字為「確認付款 NT$ 75」而非單純「購買」，再次確認金額，減少事後爭議。
- **消除疑慮文字**：「付款後即可在 Vine 聊天室使用」— 說明用途，消除「買了要去哪用？」的疑惑，對新用戶特別重要。

```
│                                             │
│   ┌─────────────────────────────────────┐   │
│   │   🛍️  繼續逛貼圖商店               │   │  ← 次要 CTA：延續購物旅程
│   └─────────────────────────────────────┘   │
```

**UX 說明**

- **同創作者推薦**：購買成功後展示同創作者作品，利用「我喜歡這個風格」的情緒溫度，轉換率顯著高於一般推薦。
- **即時庫存更新**：購買成功後，聊天室的貼圖面板應即時反映新增的貼圖組（WebSocket 推送），用戶切換到聊天室時不需重啟 App。

```
└─────────────────────────────────────────────┘

最後一個 Tab（🏷+）展開後：

│  ┌────┐ ┌────┐ ┌────┐                      │
│  │[鎖定]│[鎖定]│[鎖定]│  NT$75 NT$45 NT$60  │  ← 未購貼圖預覽，點擊跳轉商品頁
│  └────┘ └────┘ └────┘                      │
```

**UX 說明**

- **Tab 以封面縮圖識別**：每個貼圖組用封面縮圖作為 Tab 圖示（LINE 做法），比文字 Tab 節省空間，視覺辨識更快。
- **面板內搜尋**：在貼圖面板內可直接搜尋「關鍵字」找到對應貼圖，適合貼圖數量多的重度用戶（LINE 有此功能）。
- **未購貼圖預覽 Tab**：最後一個 Tab 展示用戶「追蹤的創作者新貼圖」或「平台推薦貼圖」的預覽（加鎖狀態），點擊直達商店購買。這個設計讓聊天室本身成為轉換漏斗的入口，是 LINE 沒有強化的機會。
- **探索更多入口**：面板底部固定「去商店」按鈕，任何時候都能從聊天室跳轉商店，不需退出聊天。

```
│  │        圖有更高的對話實用性。                     │  │

│  └────┘ └────┘ └────┘ └────┘                          │

│  如果你認為審核有誤，可在 5 個工作天內提出申訴。       │
```

**UX 說明**

- **結構化拒絕原因**：拒絕原因分「類別 + 問題描述 + 建議改善方向」三層。LINE 的拒絕通知非常簡略，常讓創作者不知如何修改。這是最重要的創作者體驗差異。
- **問題圖檔標注**：若審核員認定特定幾張圖有問題，直接列出圖號，讓創作者精確知道要修改哪幾張，而非重繪整組。
- **申訴倒計時**：申訴期限清楚顯示倒計時，製造行動緊迫感，避免創作者忘記申訴時限。
- **申訴 vs 直接修改**：提供兩條路徑：不服裁決 → 申訴；接受 → 直接修改重提。讓創作者自主選擇策略。

## 整體 UX 流程關鍵差異（vs LINE Creators Market）

| 面向 | LINE Creators Market | Vine（設計目標）|
|------|---------------------|-----------------|
| 貼圖定價顯示 | LINE Coins（需心算換算） | 直接顯示真實貨幣 |
| 創作者分潤透明度 | 不在介面中說明 | 定價頁即時顯示預估分潤 |
| 上傳驗證時機 | 提交後才報錯 | 上傳後即時逐張驗證 |
| 拒絕原因說明 | 簡略（1–2 行） | 類別 + 描述 + 建議 + 問題圖標注 |
| 審核進度透明度 | 無時程預估 | 顯示預計完成天數 |
| 用戶評價機制 | 無 | 有（評分 + 短評）|
| 購買後引導 | 確認按鈕 | 雙 CTA（去用 / 繼續逛）|
| 創作者追蹤 | 無 | 有（追蹤 + 新品通知）|
| 聊天室推薦入口 | 弱 | 面板內嵌未購貼圖預覽 |
| 報表深度 | 銷售數 + 金額 | + 趨勢圖 + 地區分布 |
| 提領費用透明度 | 申請後才揭露 | 申請前完整試算明細 |
