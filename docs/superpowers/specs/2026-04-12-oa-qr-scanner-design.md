# OA QR Code Scanner Design

## Problem

The QR code icon on the main page is a placeholder (shows toast). We need to implement QR code scanning that resolves an OA and displays its detail view.

## QR Code Content Format

Two formats supported, auto-detected by the scanner:

| Format | Example | Parsing |
|--------|---------|---------|
| Vine URL | `https://vine.app/oa/@my-oa` | Regex extract `/oa/(.+)` |
| Raw unique_id | `@my-oa` | Direct use |

The QR code content is always a Vine URL (`https://vine.app/oa/{unique_id}`) when generated by Vine. Raw `unique_id` is supported as a fallback for manually entered IDs or future interop.

## Architecture

Two paths from QR scan result:

```
External scan (phone camera → browser)
  → Opens https://vine.app/oa/@my-oa
  → /oa/[oaId] route page renders OA detail

Internal scan (Vine app scanner)
  → parseOAScanResult(content) → unique_id
  → ResolveOfficialAccount({ unique_id })
  → OADetailSheet opens
```

## New Files

### 1. `apps/web/src/features/oa/parse-qr.ts`

Shared parsing utility:

```ts
export function parseOAScanResult(content: string): string {
  // Match: https://vine.app/oa/@my-oa
  const urlMatch = content.match(/vine\.app\/oa\/(.+)/)
  if (urlMatch) return decodeURIComponent(urlMatch[1])
  // Fallback: treat as raw unique_id
  return content
}
```

### 2. `apps/web/src/interface/dialogs/OAScannerSheet.tsx`

QR scanner sheet component:
- Opens as a modal Sheet (same pattern as MainHeaderMenu)
- Uses a QR scanning library (platform-dependent)
- On scan success: `parseOAScanResult` → `ResolveOfficialAccount` → close scanner → open OADetailSheet
- On scan error / invalid content: showToast error

### 3. `apps/web/app/(app)/oa/[oaId].tsx`

Deep link route page for external QR scans:
- `useActiveParams<{ oaId: string }>` to get the unique_id
- Fetches OA via `ResolveOfficialAccount({ unique_id: oaId })`
- Renders `OADetailContent` component (shared with Sheet)
- Handles logged-in and logged-out states:
  - Logged in: full OA detail with actions
  - Logged out: OA info preview + login prompt

### 4. `apps/web/src/interface/oa/OADetailContent.tsx`

Extracted shared content component from OADetailSheet:
- Contains all the visual content (cover, logo, badge, description, actions, social, etc.)
- Receives OA data + friendship state + action callbacks as props
- Used by both `OADetailSheet` and `/oa/[oaId]` page

## Data Flow

### Internal Scanner Flow

```
User taps QR icon
  → OAScannerSheet opens
  → Camera activates for QR scanning
  → Scan result: "https://vine.app/oa/@my-oa"
  → parseOAScanResult → "@my-oa"
  → oaClient.resolveOfficialAccount({ uniqueId: "@my-oa" })
  → Response: { id, name, uniqueId, description, imageUrl }
  → Scanner closes
  → openDetail({ id, name, oaId: uniqueId, imageUrl })
  → OADetailSheet renders
```

### External Deep Link Flow

```
User scans QR with phone camera
  → Browser opens https://vine.app/oa/@my-oa
  → One router matches /oa/[oaId]
  → oaId = "@my-oa"
  → OADetailPage fetches OA data
  → Renders OADetailContent
```
